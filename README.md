## Eligibility criteria for a mentor

### Project

**`¬´A1¬ª`** At the root of the project there is a folder `images` with images.

**`¬´A2¬ª`** At the root of the project there is a `css` folder with style files.

**`¬´A3¬ª`** All styles are written in one `styles.css` file, which is located in the `css` folder.

**`¬´A4¬ª`** There are no capital letters, spaces and transliteration in the file names, only letters and words of the English language.

**`¬´A5¬ª`** Source code formatted with `Prettier`.

**`¬´A6¬ª`** All images and text content are taken from the layout.

**`¬´A7¬ª`** All HTML pages have a style normalizer [`modern-nomalize`](https://github.com/sindresorhus/modern-normalize).

**`¬´A8¬ª`** The code is written as follows [**manual**](https://codeguide.co/).

### Formalization

**`¬´B1¬ª`** Allow global reset of styles by tag selector for elements `<h1>...<h6>`, `<p>` –∏ `<ul>`.

**`¬´B2¬ª`** Elements do not have margins (the `margin` property) that "pierce" parent element.

**`¬´B3¬ª`** In single-line element collections, cleared the leftmost or rightmost `margin` elements (if any).

**`¬´B4¬ª`** For padding between two adjacent elements, use the property `margin`.

**`¬´B5¬ª`** For the gap between the border of the parent and its child, use the `padding` property.

**`¬´B6¬ª`** Sizes of outer margins (`margin` property) and inner margins (the `padding` property) of the elements are set exactly according to the layout.

**`¬´B7¬ª`** Generic `.container` helper class created for centering and content width restrictions.

**`¬´B8¬ª`** The width of the "container" corresponds to the layout and is equal to `1200px`.

**`¬´B9¬ª`** "Container" wraps the content of the header, footer and sections. That is inside them.

**`¬´B10¬ª`** `Flexbox` is used to position elements, but only where necessary. For example, in the header, navigation, lists in sections, and so on, that is, where it is impossible to arrange elements horizontally differently.

**`¬´B11¬ª`** The final sizes of the blocks in the browser correspond to the layout.

**`¬´B12¬ª`** Elements do not have a fixed height, it is determined by their content.

**`¬´B13¬ª`** The header has a bottom frame, you need to zoom in very close to the layout to see it.

**`¬´B14¬ª`** The sections are stacked on top of each other like a stack of books, with no outside padding.

**`¬´B15¬ª`** For all sections, one `.section` class is used, top and bottom paddings are set by `94px`, pushing the content inside the section.

**`¬´B16¬ª`** To build grids, the technique described in the abstract and video workshop is used.

**`¬´B17¬ª`** The cards on the `Portfolio` page have a border (the `border` property), but only at the bottom of the card.
